Dit document beschrijft een applicatieprofiel, in dit geval Besluitvorming Vlaamse Regering. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Samenvatting

Status van dit document

Dit applicatieprofiel dient om het specifieke gebruik van de entiteiten relevant voor de beschreven applicatie te verduidelijken.

Dit document werd als Kandidaat-standaard gepubliceerd door de werkgroep semantiek en bevindt zich in een evaluatieperiode. Indien de evaluatie positief beoordeeld wordt door het stuurorgaan Vlaams Informatie- en ICT-beleid op basis van een aantal referentie implementaties en het behandelen van eventuele feedback wordt dit document gepromoot tot Standaard in het beschreven domein.

Feedback op deze specificatie kan gegeven worden via de mailing lijst of als een topic in onze publieke reviewdiscussielijst.

Licentie

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

Entiteiten

Agenda

Beschrijving
Agenda die besproken wordt tijdens een zitting.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bevat, datum uitgestuurd, finale versie, heeft vorige versie, opmerking, versie.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bevat Agendapunt 0...* Duidt aan dat een agenda uit meerdere agendapunten kan bestaan.
datum uitgestuurd DateTime 1 De datum waarop een agenda werd uitgestuurd.
finale versie Boolean 1 De indicator die aangeeft of een agenda de finale versie is voor een zitting.
heeft vorige versie Agenda 0...1 Duidt aan dat een agenda een vorige versie kan hebben.
opmerking Opmerking 0...* De opmerking gegeven bij een agenda.
versie Versie 1 De indicator die de versie van een agenda aangeeft, bv. agenda A, agenda B, enzovoort.

Agendapunt

Beschrijving
Formeel aanvaard te behandelen onderwerp op een zitting.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanmaakdatum, formeel goedgekeurd, geeft aanleiding tot, heeft aanwezige, ingetrokken, is aangebracht na, notulen, opmerking, titel persagenda, verdaagd.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanmaakdatum DateTime 1 Het tijdstip waarop een agendapunt is aangemaakt.
formeel goedgekeurd Boolean 1 De indicator die aangeeft of een agendapunt formeel goedgekeurd is.
geeft aanleiding tot Beslissing 0...1 Duidt aan dat een besluit als gevolg van een agendapunt aangemaakt kan worden.
heeft aanwezige Agent 0...* Duidt aan dat een agent aanwezig is bij de behandeling van een agendapunt.
ingetrokken Boolean 1 De indicator die aangeeft of een agendapunt is ingetrokken.
is aangebracht na Agendapunt 0...1 Duidt aan dat een agendapunt na een ander agendapunt is aangebracht op een zitting.
notulen String 1 De formele notulen van een zitting.
opmerking Opmerking 0...* De opmerking gegeven bij een agendapunt.
titel persagenda String 1 De titel van een agendapunt in de persagenda.
verdaagd Verdaagd 1 De indicator die aangeeft of een agendapunt verdaagd is en eventueel naar welke zitting.

Agent

Beschrijving
Iemand die of iets dat kan handelen of een effect kan teweeg brengen.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: beleidsdomein, bevoegheid.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beleidsdomein Beleidsdomein 0...* Beleidsdomein van de Vlaamse Overheid, bestaande uit een departement en één of meer agentschappen.
bevoegheid Bevoegdheid 0...* De toestemming om een handeling te mogen verrichten.

Beslissing

Beschrijving
Authentieke schriftelijke neerslag van een beslissing van een bestuursorgaan. Deze beslissing houdt een rechtshandeling in waarbij sprake is van een beoogd rechtsgevolg.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: beschrijving, citeeropschrift, geeft aanleiding tot, goedgekeurd, is gerealiseerd door.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 1 De beknopte beschrijving van de beslissing.
citeeropschrift String 1 De beknopte titel of officiële korte naam van een decreet, wet, besluit... Deze wordt officieel vastgelegd. Deze benaming wordt in de praktijk gebruikt om naar de rechtsgrond te verwijzen.
geeft aanleiding tot NieuwsbriefInfo 0...1 Duidt aan dat een nieuwsbriefinfo als gevolg van een besluit aangemaakt kan worden.
goedgekeurd Boolean 1 De indicator die aangeeft of een beslissing is goedgekeurd.
is gerealiseerd door Publicatie 0...* Duidt aan dat een beslissing gerealiseerd wordt via een publicatie.

Beschrijving
Input gegeven door een welbepaald orgaan.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bevat, ontvangstdatum, opmerking, samenvatting, uitkomst.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bevat Document 0...* Duidt aan dat een documentversie onderdeel kan uitmaken van een consultatie-antwoord.
ontvangstdatum DateTime 1 De datum waarop een consultatie-antwoord is ontvangen.
opmerking Opmerking 0...* De opmerking gegeven bij een consultatie-antwoord.
samenvatting String 1 De korte tekst die een ontvangen consultatie-antwoord samenvat.
uitkomst Consultatie-uitkomst 1 Het kernwoord dat de uitkomst van een consultatie omschrijft.

Vraag tot consultatie.
Eigenschappen

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

opmerking Opmerking 0...* De opmerking gegeven bij een consultatie-antwoord.
type ConsultatieType 1 Het type van een consulatie, bv. advies, raadpleging, etc.

Document

Beschrijving
Origineel werk in de brede zin (een verslag, een opname, een boek...).
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: gearchiveerd, heeft versie, naam, opmerking, stuknummer Vlaams Parlement, stuknummer Vlaamse Regering, type, vertrouwelijkheid.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

naam String 0...* De naam waarmee een document wordt aangeduid.
opmerking Opmerking 0...* De opmerking gegeven bij een document.

stuknummer Vlaamse Regering String 1 Het werknummer zoals het door de Vlaamse Regering is toegekend.
type DocumentType 1 De aard van een document, bijvoorbeeld: decreet, mededeling, nota etc.
vertrouwelijkheid Vertrouwelijkheid 1 De indicator die de mate van vertrouwelijkheid van een document aangeeft.

Documentversie

Beschrijving
Versie van een document.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanreikdatum, formaat, is aangebracht door, is vertaling van, naam, nummer, opmerking, taal, vereist, versie.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

nummer String 1 Het numeer dat een documentversie identificeert.
opmerking Opmerking 0...* De opmerking gegeven bij een documentversie.
taal Taal 1 De taal waarin een documentversie geschreven is. Link

versie String 1 De identificator van een specifieke versie van een document.

Dossier

Beschrijving
Overkoepelend geheel van documenten ter voorbereiding van één of meerdere agendapunten, die verschillende procedurestappen doorloopt.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aanmaakdatum, doorloopt, heeft bevoegde, heeft contactpersoon, heeft creator, heeft indiener, is gerelateerd aan, korte titel, nummer, opmerking, thema, titel.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aanmaakdatum DateTime 1 Het tijdstip waarop een dossier is aangemaakt.
doorloopt Procedurestap 0...* Duidt aan dat een dossier verschillende stappen doorloopt, waarbij een procedurestap een welbepaalde fase beschrijft.
heeft bevoegde Agent 1...* Duidt aan dat een dossier één of meer agenten als bevoegde heeft.
heeft contactpersoon Agent 1...* Duidt aan dat een dossier één of meerdere agenten als contactpersoon kan hebben.
heeft creator Agent 1 Duidt aan dat een dossier door een agent is aangemaakt.
heeft indiener Agent 1 Duidt aan dat een dossier een agent als indiener heeft.
is gerelateerd aan Dossier 0...* Duidt aan dat een dossier gerelateerd kan zijn aan een ander dossier.
korte titel String 1 De korte titel van een dossier.
nummer String 1 Het nummer dat een dossier identificeert.
opmerking Opmerking 0...* De opmerking gegeven bij een dossier.
thema Thema 1...* Het thema van een dossier.
titel String 1 De titel van een dossier.

Procedurestap

Beschrijving
Formeel gedefinieerde stappen die in de juiste volgorde leiden tot de afronding van een dossier.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bevat, bevat, fase, geeft aanleiding tot, is aangevraagd voor, is geagendeerd via, is gerelateerd aan, korte titel, opmerking, titel, type, vertrouwelijkheid.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bevat Documentversie 0...* Duidt aan dat een dossier in een welbepaalde procedurestap een documentversie kan bevatten.
bevat Consultatievraag 0...* Duidt aan dat een groter geheel uit meerdere onderdelen kan bestaan.
fase Status 1...* De status die een procedurestap beschrijft, bv. eerste principiële goedkeuring.
geeft aanleiding tot Beslissing 0...1 Duidt aan dat een besluit als gevolg van een dossier in een welbepaalde procedurestap aangemaakt kan worden.
is aangevraagd voor Zitting 1 Duidt aan dat iemand heeft aangevraagd om een procedurestap op de agenda te plaatsen van een welbepaalde zitting.
is geagendeerd via Agendapunt 0...* Duidt aan dat een dossier in een welbepaalde procedurestap gebruikt wordt als informatiebron om een agendapunt te bespreken.
is gerelateerd aan Procedurestap 0...* Duidt aan dat een procedurestap gerelateerd kan zijn aan een andere procedurestap.
korte titel String 1 De verkorte titel van een dossier in een welbepaalde procedurestap.
opmerking Opmerking 0...* De opmerking gegeven bij een dossier in een welbepaalde procedurestap.
titel String 1 De titel van een dossier in een welbepaalde procedurestap.
type ProcedurestapType 1 Het type van een procedurestap. Dit is ofwel een aktename ofwel een beslissing.
vertrouwelijkheid Vertrouwelijkheid 1 De indicator die de mate van vertrouwelijkheid van een procedurestap aangeeft.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

geëindigd op tijdstip DateTime 1 Het tijdstip waarop een zitting eindigt.
nummer Integer 1 Het nummer dat een ministerraad identificeert.
