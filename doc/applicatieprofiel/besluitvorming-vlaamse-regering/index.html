Dit document beschrijft een applicatieprofiel, in dit geval Besluitvorming Vlaamse Regering. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Samenvatting

Status van dit document

Dit applicatieprofiel dient om het specifieke gebruik van de entitieten relevant voor de beschreven applicatie te verduidelijken.

Dit document werd als Kandidaat-standaard gepubliceerd door de werkgroep semantiek en bevindt zich in een evaluatieperiode. Indien de evaluatie positief beoordeeld wordt door het stuurorgaan Vlaams Informatie- en ICT-beleid op basis van een aantal referentie implementaties en het behandelen van eventuele feedback wordt dit document gepromoot tot Standaard in het beschreven domein.

Feedback op deze specificatie kan gegeven worden via de mailing lijst of als een topic in onze publieke reviewdiscussielijst.

Licentie

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

datum van ontvangst Date 1 De datum waarop het advies werd ontvangen.

Agendapunt

Beschrijving

Eigenschappen

datum waarnaar uitgesteld Date 1 De datum naar waar een agendapunt werd uitgesteld.
formeel goedgekeurd Boolean 1 Een indicator die aangeeft of een agendapunt formeel goedgekeurd is.
leidt tot KortBestek 1 Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.
leidt tot Beslissing 0...1 Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.
notulen String 1 De formele notulen van een zitting.
opmerking String 0...* Een opmerking gegeven bij een agendapunt.
persagenda boolean 1 Een indicator die aangeeft of een agendapunt in de persagenda mag verschijnen.
vertrouwelijk Vertrouwelijk 1 Een indicator die de mate van vertrouwelijkheid van een agendapunt aangeeft.

Deeldossier

Beschrijving
Formeel gedefiniëerde stappen die in de juiste volgorde leiden tot de afronding van een dossier. Deze volgorde kan zowel (deels) sequentieel als parallel gedefiniëerd zijn.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: behandelingsstatus, bevat, datum aangevraagd voor ministerraad, datum ingepland voor ministerraad, datum verdaagd naar ministerraad, is geagendeerd via, korte titel, leidt tot, opmerking, titel, type van dossier.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
behandelingsstatus Behandelingsstatus 1 De status waarin een deeldossier zich bevindt.
bevat Advies 0...* Duidt aan dat bepaalde objecten onderdeel kunnen uitmaken van een groter geheel.
datum aangevraagd voor ministerraad Date 1 De datum van de ministerraad waarop men een deeldossier wil agenderen.
datum ingepland voor ministerraad Date 1 De datum van de ministerraad waarop een deeldossier is ingepland om te bespreken.
datum verdaagd naar ministerraad Date 1 De datum van de ministerraard naar waar een deeldossier is verdaagd.
is geagendeerd via Agendapunt 0...1 Duidt aan dat een deeldossier gebruikt wordt als informatiebron om een agendapunt te bespreken.
korte titel String 1 De verkorte titel van een deeldossier.
leidt tot Beslissing 0...* Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.
opmerking String 0...* Een opmerking gegeven bij een deeldossier.
titel String 1 De titel van een deeldossier.
type van dossier TypeDossier 1 Het type van een deeldossier.
