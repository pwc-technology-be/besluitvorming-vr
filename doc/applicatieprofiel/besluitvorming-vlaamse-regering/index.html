Dit document beschrijft een applicatieprofiel, in dit geval Besluitvorming Vlaamse Regering. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Samenvatting

Status van dit document

Dit applicatieprofiel dient om het specifieke gebruik van de entitieten relevant voor de beschreven applicatie te verduidelijken.

Dit document werd als Kandidaat-standaard gepubliceerd door de werkgroep semantiek en bevindt zich in een evaluatieperiode. Indien de evaluatie positief beoordeeld wordt door het stuurorgaan Vlaams Informatie- en ICT-beleid op basis van een aantal referentie implementaties en het behandelen van eventuele feedback wordt dit document gepromoot tot Standaard in het beschreven domein.

Feedback op deze specificatie kan gegeven worden via de mailing lijst of als een topic in onze publieke reviewdiscussielijst.

Licentie

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

Entiteiten

Advies

Beschrijving
Advies gegeven door een welbepaald orgaan met betrekking tot een dossier in een welbepaalde procedurestap.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: advies samenvatting, adviesaanvraag, adviesorgaan, adviesuitkomst, leidt tot, status.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
advies samenvatting String 1 De korte tekst die een ontvangen advies samenvat.
adviesaanvraag String 1 De tekstuele inhoud van een adviesaanvraag.
adviesorgaan String 1 Het orgaan dat instaat voor het verstrekken van een advies.
adviesuitkomst Adviesuitkomst 1 Het kernwoord dat de uitkomst van een advies beschrijft.
leidt tot Documentversie 0...* Duidt aan dat een object als gevolg van een ander object ontvangen kan worden.
status Status 1 De status waarin een advies zich bevindt, bv. aangevraagd.

geeft aanleiding tot KortBestek 1 Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.
geeft aanleiding tot Besluit 0...1 Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.

opmerking String 0...* Een opmerking gegeven bij een agendapunt.
persagenda Boolean 1 De indicator die aangeeft of een agendapunt in de persagenda mag verschijnen.
vertrouwelijk Vertrouwelijk 1 Een indicator die de mate van vertrouwelijkheid van een agendapunt aangeeft.

heeft bevoegde Hoedanigheid 0...* Duidt aan dat een dossier één of meer hoedanigheden als bevoegde heeft.
heeft contactpersoon Hoedanigheid 0...1 Duidt aan dat een entiteit een hoedanigheid als contactpersoon heeft.
heeft indiener Hoedanigheid 0...1 Duidt aan dat een dossier een hoedanigheid als indiener heeft.
korte titel String 1 De verkorte titel van een dossier.
nummer Integer 1 Het nummer dat een dossier identificeert.
opmerking String 0...* Een opmerking gegeven bij een dosier.
thema Thema 0...* Het thema van een dossier.
titel String 1 De titel van een dossier.

Procedurestap

Beschrijving
Formeel gedefinieerde stappen die in de juiste volgorde leiden tot de afronding van een dossier.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bevat, datum aangevraagd voor ministerraad, datum verdaagd naar ministerraad, fase, geeft aanleiding tot, is geagendeerd via, korte titel, opmerking, titel, type van dossier.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bevat Advies 0...* Duidt aan dat een groter geheel uit meerdere onderdelen kan bestaan.
datum aangevraagd voor ministerraad Date 0...1 De datum van de ministerraad waarop een dossier, dat zich in een welbepaalde procedurestap bevindt, is geagendeerd.
datum verdaagd naar ministerraad Date 0...1 De datum van de ministerraard waarnaar een dossier, dat zich in een welbepaalde procedurestap bevindt, is verdaagd.
fase Fase 1 De status die een procedurestap beschrijft, bv. eerste principiële goedkeuring.
geeft aanleiding tot Besluit 0...* Duidt aan dat een object als gevolg van een ander object aangemaakt kan worden.
is geagendeerd via Agendapunt 0...1 Duidt aan dat een dossier in een welbepaalde procedurestap gebruikt wordt als informatiebron om een agendapunt te bespreken.
korte titel String 1 De verkorte titel van een dossier in een welbepaalde procedurestap.
opmerking String 0...* Een opmerking gegeven bij een dossier in een welbepaalde procedurestap.
titel String 1 De titel van een dossier in een welbepaalde procedurestap.
type van dossier TypeDossier 1 Het type van een dossier in een welbepaalde procedurestap.
